56  /***
57   * A simple service-provider lookup mechanism.  A <i>service</i> is a
58   * well-known set of interfaces and (usually abstract) classes.  A <i>service
59   * provider</i> is a specific implementation of a service.  The classes in a
60   * provider typically implement the interfaces and subclass the classes defined
61   * in the service itself.  Service providers may be installed in an
62   * implementation of the Java platform in the form of extensions, that is, jar
63   * files placed into any of the usual extension directories.  Providers may
64   * also be made available by adding them to the applet or application class
65   * path or by some other platform-specific means.
66   * <p/>
67   * <p> In this lookup mechanism a service is represented by an interface or an
68   * abstract class.  (A concrete class may be used, but this is not
69   * recommended.)  A provider of a given service contains one or more concrete
70   * classes that extend this <i>service class</i> with data and code specific to
71   * the provider.  This <i>provider class</i> will typically not be the entire
72   * provider itself but rather a proxy that contains enough information to
73   * decide whether the provider is able to satisfy a particular request together
74   * with code that can create the actual provider on demand.  The details of
75   * provider classes tend to be highly service-specific; no single class or
76   * interface could possibly unify them, so no such class has been defined.  The
77   * only requirement enforced here is that provider classes must have a
78   * zero-argument constructor so that they may be instantiated during lookup.
79   * <p/>
80   * <p> A service provider identifies itself by placing a provider-configuration
81   * file in the resource directory <tt>META-INF/services</tt>.  The file's name
82   * should consist of the fully-qualified name of the abstract service class.
83   * The file should contain a list of fully-qualified concrete provider-class
84   * names, one per line.  Space and tab characters surrounding each name, as
85   * well as blank lines, are ignored.  The comment character is <tt>'#'</tt>
86   * (<tt>0x23</tt>); on each line all characters following the first comment
87   * character are ignored.  The file must be encoded in UTF-8.
88   * <p/>
89   * <p> If a particular concrete provider class is named in more than one
90   * configuration file, or is named in the same configuration file more than
91   * once, then the duplicates will be ignored.  The configuration file naming a
92   * particular provider need not be in the same jar file or other distribution
93   * unit as the provider itself.  The provider must be accessible from the same
94   * class loader that was initially queried to locate the configuration file;
95   * note that this is not necessarily the class loader that found the file.
96   * <p/>
97   * <p> <b>Example:</b> Suppose we have a service class named
98   * <tt>java.io.spi.CharCodec</tt>.  It has two abstract methods:
99   * <p/>
100  * <pre>
101  *   public abstract CharEncoder getEncoder(String encodingName);
102  *   public abstract CharDecoder getDecoder(String encodingName);
103  * </pre>
104  * <p/>
105  * Each method returns an appropriate object or <tt>null</tt> if it cannot
106  * translate the given encoding.  Typical <tt>CharCodec</tt> providers will
107  * support more than one encoding.
108  * <p/>
109  * <p> If <tt>sun.io.StandardCodec</tt> is a provider of the <tt>CharCodec</tt>
110  * service then its jar file would contain the file
111  * <tt>META-INF/services/java.io.spi.CharCodec</tt>.  This file would contain
112  * the single line:
113  * <p/>
114  * <pre>
115  *   sun.io.StandardCodec    # Standard codecs for the platform
116  * </pre>
117  * <p/>
118  * To locate an codec for a given encoding name, the internal I/O code would
119  * do something like this:
120  * <p/>
121  * <pre>
122  *   CharEncoder getEncoder(String encodingName) {
123  *       for( CharCodec cc : ServiceFinder.find(CharCodec.class) ) {
124  *           CharEncoder ce = cc.getEncoder(encodingName);
125  *           if (ce != null)
126  *               return ce;
127  *       }
128  *       return null;
129  *   }
130  * </pre>
131  * <p/>
132  * The provider-lookup mechanism always executes in the security context of the
133  * caller.  Trusted system code should typically invoke the methods in this
134  * class from within a privileged security context.
135  *
136  * @author Mark Reinhold
137  * @version 1.11, 03/12/19
138  * @since 1.3
139  */
140 final class ServiceFinder<T> implements Iterable<T> {

142     private static final String prefix = "META-INF/services/";

144     private final Class<T> serviceClass;
145     private final @Nullable
146     ClassLoader classLoader;

148     /***
149      * Locates and incrementally instantiates the available providers of a
150      * given service using the given class loader.
151      * <p/>
152      * <p> This method transforms the name of the given service class into a
153      * provider-configuration filename as described above and then uses the
154      * <tt>getResources</tt> method of the given class loader to find all
155      * available files with that name.  These files are then read and parsed to
156      * produce a list of provider-class names.  The iterator that is returned
157      * uses the given class loader to lookup and then instantiate each element
158      * of the list.
159      * <p/>
160      * <p> Because it is possible for extensions to be installed into a running
161      * Java virtual machine, this method may return different results each time
162      * it is invoked. <p>
163      *
164      * @param service The service's abstract service class
165      * @param loader  The class loader to be used to load provider-configuration files
166      *                and instantiate provider classes, or <tt>null</tt> if the system
167      *                class loader (or, failing that the bootstrap class loader) is to
168      *                be used
169      * @throws ServiceConfigurationError If a provider-configuration file violates the specified format
170      *                                   or names a provider class that cannot be found and instantiated
171      * @see #find(Class)
172      */
173     public static <T> ServiceFinder<T> find(@NotNull Class<T> service, @Nullable ClassLoader loader) {
174         return new ServiceFinder<T>(service,loader);
175     }

177     /***
178      * Locates and incrementally instantiates the available providers of a
179      * given service using the context class loader.  This convenience method
180      * is equivalent to
181      * <p/>
182      * <pre>
183      *   ClassLoader cl = Thread.currentThread().getContextClassLoader();
184      *   return Service.providers(service, cl);
185      * </pre>
186      *
187      * @param service The service's abstract service class
188      *
189      * @throws ServiceConfigurationError If a provider-configuration file violates the specified format
190      *                                   or names a provider class that cannot be found and instantiated
191      * @see #find(Class, ClassLoader)
192      */
193     public static <T> ServiceFinder<T> find(Class<T> service) {
194         return find(service,Thread.currentThread().getContextClassLoader());
195     }

197     private ServiceFinder(Class<T> service, ClassLoader loader) {
198         this.serviceClass = service;
199         this.classLoader = loader;
200     }

202     /***
203      * Returns discovered objects incrementally.
204      *
205      * @return An <tt>Iterator</tt> that yields provider objects for the given
206      *         service, in some arbitrary order.  The iterator will throw a
207      *         <tt>ServiceConfigurationError</tt> if a provider-configuration
208      *         file violates the specified format or if a provider class cannot
209      *         be found and instantiated.

212         return new LazyIterator<T>(serviceClass,classLoader);

215     /***
216      * Returns discovered objects all at once.
217      *
218      * @return
219      *      can be empty but never null.
220      *
221      * @throws ServiceConfigurationError
222      */
223     public T[] toArray() {
224         List<T> result = new ArrayList<T>();
225         for (T t : this) {
226             result.add(t);
227         }
228         return result.toArray((T[]) Array.newInstance(serviceClass,result.size()));

249     /***
250      * Parse a single line from the given configuration file, adding the name
251      * on the line to both the names list and the returned set iff the name is
252      * not already a member of the returned set.
253      */
254     private static int parseLine(Class service, URL u, BufferedReader r, int lc,
255                                  List<String> names, Set<String> returned)
256         throws IOException, ServiceConfigurationError {
257         String ln = r.readLine();
258         if (ln == null) {
259             return -1;
260         }
261         int ci = ln.indexOf('#');
262         if (ci >= 0) ln = ln.substring(0, ci);
263         ln = ln.trim();
264         int n = ln.length();
265         if (n != 0) {
266             if ((ln.indexOf(' ') >= 0) || (ln.indexOf('\t') >= 0))
267                 fail(service, u, lc, "Illegal configuration-file syntax");
268             int cp = ln.codePointAt(0);
269             if (!Character.isJavaIdentifierStart(cp))
270                 fail(service, u, lc, "Illegal provider-class name: " + ln);
271             for (int i = Character.charCount(cp); i < n; i += Character.charCount(cp)) {
272                 cp = ln.codePointAt(i);
273                 if (!Character.isJavaIdentifierPart(cp) && (cp != '.'))
274                     fail(service, u, lc, "Illegal provider-class name: " + ln);
275             }
276             if (!returned.contains(ln)) {
277                 names.add(ln);
278                 returned.add(ln);
279             }
280         }
281         return lc + 1;
282     }

284     /***
285      * Parse the content of the given URL as a provider-configuration file.
286      *
287      * @param service  The service class for which providers are being sought;
288      *                 used to construct error detail strings
289      * @param u        The URL naming the configuration file to be parsed
290      * @param returned A Set containing the names of provider classes that have already
291      *                 been returned.  This set will be updated to contain the names
292      *                 that will be yielded from the returned <tt>Iterator</tt>.
293      * @return A (possibly empty) <tt>Iterator</tt> that will yield the
294      *         provider-class names in the given configuration file that are
295      *         not yet members of the returned set
296      * @throws ServiceConfigurationError If an I/O error occurs while reading from the given URL, or
297      *                                   if a configuration-file format error is detected
298      */
299     @SuppressWarnings({"StatementWithEmptyBody"})
300     private static Iterator<String> parse(Class service, URL u, Set<String> returned)
301         throws ServiceConfigurationError {
302         InputStream in = null;
303         BufferedReader r = null;
304         ArrayList<String> names = new ArrayList<String>();
305         try {
306             in = u.openStream();
307             r = new BufferedReader(new InputStreamReader(in, "utf-8"));
308             int lc = 1;
309             while ((lc = parseLine(service, u, r, lc, names, returned)) >= 0) ;
310         } catch (IOException x) {
311             fail(service, ": " + x);
312         } finally {
313             try {
314                 if (r != null) r.close();
315                 if (in != null) in.close();
316             } catch (IOException y) {
317                 fail(service, ": " + y);
318             }
319         }
320         return names.iterator();
321     }

324     /***
325      * Private inner class implementing fully-lazy provider lookup
326      */
327     private static class LazyIterator<T> implements Iterator<T> {
328         Class<T> service;
329         @Nullable ClassLoader loader;
330         Enumeration<URL> configs = null;
331         Iterator<String> pending = null;
332         Set<String> returned = new TreeSet<String>();
333         String nextName = null;
334         URL currentConfig = null;
335 
336         private LazyIterator(Class<T> service, ClassLoader loader) {
337             this.service = service;
338             this.loader = loader;
339         }
340 
341         public boolean hasNext() throws ServiceConfigurationError {
342             if (nextName != null) {
343                 return true;
344             }
345             if (configs == null) {
346                 try {
347                     String fullName = prefix + service.getName();
348                     if (loader == null)
349                         configs = ClassLoader.getSystemResources(fullName);
350                     else
351                         configs = loader.getResources(fullName);
352                 } catch (IOException x) {
353                     fail(service, ": " + x);
354                 }
355             }
356             while ((pending == null) || !pending.hasNext()) {
357                 if (!configs.hasMoreElements()) {
358                     return false;
359                 }
360                 currentConfig = configs.nextElement();
361                 pending = parse(service, currentConfig, returned);
362             }
363             nextName = pending.next();
364             return true;
365         }
366 
367         public T next() throws ServiceConfigurationError {
368             if (!hasNext()) {
369                 throw new NoSuchElementException();
370             }
371             String cn = nextName;
372             nextName = null;
373             try {
374                 return service.cast(Class.forName(cn, true, loader).newInstance());
375             } catch (ClassNotFoundException x) {
376                 fail(service,
377                     "Provider " + cn + " is specified in "+currentConfig+" but not found");
378             } catch (Exception x) {
379                 fail(service,
380                     "Provider " + cn + " is specified in "+currentConfig+"but could not be instantiated: " + x, x);
381             }
382             return null;    /* This cannot happen */
383         }
384 
385         public void remove() {
386             throw new UnsupportedOperationException();
387         }
388     }
