hk2:run

Launches the module system by using all the dependencies as modules.

Not specifying "@execute phase=compile" so that it runs quickly even with a large project.

Mojo Attributes:

  • Requires a Maven 2.0 project to execute.
  • Requires dependency resolution of artifacts in scope: runtime
  • Automatically executes within the lifecycle phase: compile

Optional Parameters

Name Type Description
args String[] Command-line options to be passed to {@link StartupContext}.
artifactFactory ArtifactFactory No description.
artifactResolver ArtifactResolver No description.
localRepository ArtifactRepository No description.
mainModule String groupId:artifactId of the module that includes {@link ModuleStartup}. If omitted, the module list is searched to find one.
rootDir File The root directory of the launched module system. Default value is ${project.baseDir}.
skipId String groupId:artifactId of the module which the hk2:run mojo just skips executing. This hack is used to specify the "hk2:run" for a test phase execution in the parent POM for all the child modules (except the parent module itself.)
